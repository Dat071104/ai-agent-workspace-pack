# Analyze Team

Use `analyze-team/` when a request needs clarification, architectural judgment, option comparison, or major structure decisions.

This team does not rush into file generation. It reads available context, asks clarifying questions, creates 2-4 options, compares pros/cons/risks, recommends one path, and waits for user confirmation before creating major structure or rules.

Vietnamese note: Dung team nay khi can phan tich yeu cau, dua ra nhieu phuong an, va xac nhan truoc khi lam lon.

Files:

- `SKILL.md`: workflow instructions.
- `ANALYZE_PROMPT.md`: copy-paste analysis prompt.
- `OPTIONS_DEBATE_TEMPLATE.md`: option comparison format.
- `DECISION_CONFIRMATION_TEMPLATE.md`: confirmation format.

Git safety:

- Never use `git add .`.
- Do not commit unless user allowed it.
- Keep implementation logs updated in target projects.
