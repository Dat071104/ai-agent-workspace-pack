Warn me about high token/time/risk cost first.
Check git state, require clean git state, recommend a recovery branch or tag, audit cleanup candidates, classify risk, and propose a batch plan.
Wait for my confirmation before any cleanup.
Safety: never use git add ., one batch at a time, test after each batch, commit only if I explicitly allow it.
Expected output: risk warning, git state, cleanup candidates, batch plan, confirmation request.
