# Clean-Code Team

Use `clean-code-team/` only for cleanup and refactoring. This is high-risk work.

Warning:

- High token cost.
- High reasoning cost.
- Use the strongest available model.
- Cleanup can break working code.
- Require clean git state.
- Create a recovery branch or tag first.
- Audit first, then propose batch plan.
- Wait for confirmation.
- Clean one batch at a time.
- Test after each batch.
- Commit each successful batch only if the user confirmed.
