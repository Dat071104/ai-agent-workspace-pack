# Performance Audit Checklist

- [ ] Startup time is acceptable.
- [ ] Build size or bundle size is understood.
- [ ] API latency is measured or estimated.
- [ ] Heavy operations are identified.
- [ ] Repeated work or repeated network calls are avoided.
- [ ] Local hardware constraints are considered.
- [ ] Large files, datasets, or generated artifacts are not accidentally tracked.
- [ ] Caching behavior is intentional.
- [ ] Bottlenecks are documented.
- [ ] Performance tests or profiling commands are recorded.
