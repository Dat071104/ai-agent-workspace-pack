# UX Audit Checklist

- [ ] Primary user flows are clear and complete.
- [ ] Loading states exist where work may take time.
- [ ] Error states are understandable and actionable.
- [ ] Empty states explain what to do next.
- [ ] Role-specific behavior is correct.
- [ ] Labels, buttons, and navigation are clear.
- [ ] Keyboard basics work for key flows.
- [ ] Focus states are visible.
- [ ] Color contrast is acceptable.
- [ ] Mobile and desktop layouts do not overlap or clip text.
- [ ] Destructive actions require confirmation.
